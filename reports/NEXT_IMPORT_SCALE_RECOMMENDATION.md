# 下一阶段导入规模建议

## 结论

- RUN_1M_LOCAL_RECOMMENDED：NOT_RECOMMENDED

## 空间估算

按 500000 行实际 H: 数据目录大小线性估算：

| 规模 | 原始线性估算 | 1.5x 安全余量 |
| --- | ---: | ---: |
| 1000000 行 | 4.76 GiB | 7.14 GiB |
| 全量 5115735 行 | 24.35 GiB | 36.53 GiB |

同时保留 S13 H: preflight 的保守全量估算：

- 保守全量索引估算：41.75 GiB
- 建议可用空间：41.87 GiB

H: 当前剩余空间从容量角度足够继续 1000000 行和全量，但容量不是主要瓶颈。

## 时间估算

按 500000 行总墙钟耗时线性估算：

| 规模 | 线性耗时估算 |
| --- | ---: |
| 1000000 行 | 约 8.86 小时 |
| 全量 5115735 行 | 约 45.34 小时 |

实际后半段 Meilisearch task 明显变慢，真实耗时可能高于线性估算。全量本机导入可能达到 2-4 天，并且会长期占用本机磁盘和索引服务。

## 建议

- 不建议在本机继续常规 1000000 行导入，除非只是做夜间长时间压测并接受 10 小时以上运行时间。
- 不建议在本机跑全量导入。
- 建议把全量导入放到腾讯云 4 核 8GB / 160GB SSD 或更高配置，并使用 Docker Compose 和持久化 SSD 数据目录。
- 若必须继续本机验证，下一步只建议做一次 1000000 行夜间压测，不建议直接全量。
